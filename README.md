# prime-data

All prime numbers under 1,000,000,000,000 (one trillion) — 37,607,912,018 primes.

Produced by a proprietary engine developed by BeltUp LLC. Distributed as a public
dataset in LEB128 gap-encoded binary format with a Python decoder included.

---

## Download

**DOI: [10.5281/zenodo.21075867](https://doi.org/10.5281/zenodo.21075867)**

The dataset is split into 36 × 1 GB files plus a SHA-256 checksum manifest.
Download all files and the manifest from the Zenodo record, then verify before use:

```
sha256sum -c primes_1T.sha256
```

## Quick Start — Decode

**Requirements:** Python 3.8+

```bash
# Count all primes and confirm the total
python decode_primes.py primes_1T_part001.bin --count

# Print the first 20 primes
python decode_primes.py primes_1T_part001.bin | head -20

# Extract all primes under 1 billion to a text file
python decode_primes.py primes_1T_part001.bin --head 50847534 > primes_1B.txt
```

The decoder streams from any single chunk file. For full-dataset operations,
pipe chunks sequentially or use the full `primes_1T.bin` if you have it assembled.

## Format

**LEB128 unsigned varint gap encoding.**

Each prime is stored as the *gap* from the previous prime, encoded as a
variable-length unsigned integer (standard LEB128):

- If the gap fits in 7 bits (≤ 127), it costs exactly **1 byte**
- Each byte stores 7 bits of value; the high bit (0x80) means "more bytes follow"
- Near one trillion, the average prime gap is ~28 — virtually every prime costs 1 byte

This encoding compresses the dataset from ~300 GB (raw uint64) to ~37.8 GB (35.2 GiB).

**File structure:** raw binary, no header, no padding, no framing. First gap is from 0,
so the first decoded value is 2 (the first prime). The decoder handles this automatically.
